üåê GitHub Workflow for Beginners

This guide teaches you the complete workflow for contributing to projects on GitHub - from forking to getting your PR merged!

üéØ The Complete Workflow

Think of GitHub workflow like this:

  1. Fork = Make your own copy
  2. Clone = Download to your computer
  3. Branch = Create a workspace
  4. Commit = Save your work
  5. Push = Upload your changes
  6. PR = Request to merge your changes
  7. Merge = Your changes become part of the main project! üéâ

üöÄ Step-by-Step Workflow

üç¥ Step 1: Fork the Repository

What it does: Creates your own copy of the project

How to do it:

  1. Go to the project page on GitHub
  2. Click the "Fork" button (top right)
  3. Wait a few seconds
  4. Done! You now have your own copy at github.com/YOUR-USERNAME/project-name

üíª Step 2: Clone to Your Computer

What it does: Downloads your fork to work locally

How to do it:

git clone https://github.com/YOUR-USERNAME/OpenPracticeHub.git
cd OpenPracticeHub

üîó Step 3: Set Up the Original as "Upstream"

What it does: Lets you get updates from the original project

How to do it:

git remote add upstream https://github.com/ratul345/OpenPracticeHub.git
git remote -v  # Check that it worked

üåø Step 4: Create a Branch

What it does: Creates a separate workspace for your changes

How to do it:

git checkout -b my-feature-name

Branch naming examples:

  • fix-typo-in-readme
  • add-quote-about-learning
  • improve-button-styling
  • make-mobile-friendly

‚ú® Step 5: Make Your Changes

What it does: The fun part - actually improving the project!

Ideas for OpenPracticeHub:

  • Fix a typo
  • Add a quote to quotes.json
  • Improve CSS styling
  • Add a new feature
  • Make something mobile-friendly

üì¶ Step 6: Stage Your Changes

What it does: Tells Git which changes to save

How to do it:

git add filename.txt    # Add specific file
git add .              # Add everything (most common!)

üíæ Step 7: Commit Your Changes

What it does: Saves your changes with a description

How to do it:

git commit -m "Add inspiring quote about learning to code"

Good commit messages:

  • ‚úÖ "Fix typo in README.md"
  • ‚úÖ "Add motivational quote about perseverance"
  • ‚úÖ "Improve button hover animation"
  • ‚ùå "Update" (too vague!)
  • ‚ùå "Fix" (what did you fix?)

üöÄ Step 8: Push to Your Fork

What it does: Uploads your changes to your GitHub fork

How to do it:

git push origin my-feature-name

üéØ Step 9: Create a Pull Request

What it does: Asks the project maintainer to review your changes

How to do it:

  1. Go to your fork on GitHub
  2. You'll see a "Compare & pull request" button - click it!
  3. Write a clear title and description
  4. Submit the PR!

üí¨ Step 10: Respond to Feedback

What it does: Shows you're a good collaborator

How to do it:

  • Check your PR regularly for comments
  • Be friendly and open to suggestions
  • Make requested changes if needed
  • Push new commits to the same branch

üîÉ Keeping Your Fork Updated

Before starting new work, always update your fork:

git checkout main              # Switch to main branch
git fetch upstream             # Get updates from original
git merge upstream/main        # Apply updates to your fork
git push origin main           # Update your GitHub fork

üé® Common Workflow Patterns

Pattern 1: Quick Fix

git checkout main
git pull upstream main
git checkout -b fix-typo
echo "Fixed!" >> README.md
git add .
git commit -m "Fix typo in README"
git push origin fix-typo
# Create PR on GitHub

Pattern 2: Feature Development

git checkout main
git pull upstream main
git checkout -b add-cool-feature
# Work on your feature...
git add .
git commit -m "Add cool feature"
git push origin add-cool-feature
# Create PR on GitHub

üÜò Common Workflow Problems

"My fork is out of sync!"

git checkout main
git fetch upstream
git reset --hard upstream/main
git push origin main --force

"I committed to the wrong branch!"

git checkout correct-branch
git cherry-pick COMMIT-HASH
git checkout wrong-branch
git reset --hard HEAD~1

"I want to start over!"

git checkout main
git branch -D my-branch-name  # Delete branch locally
git push origin --delete my-branch-name  # Delete on GitHub

üöÄ Pro Tips

  • Commit often: Small commits are easier to review
  • Write good messages: Help reviewers understand your changes
  • Test before committing: Make sure your changes work
  • Be patient: Reviews take time - maintainers are volunteers
  • Stay friendly: Open source is about collaboration, not competition
