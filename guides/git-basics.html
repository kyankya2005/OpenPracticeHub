üéØ What is Git?

Git is like a "save button" for your code, but way more powerful. It tracks every change you make, lets you go back in time, and helps you collaborate with others.

Think of it like this:

  • Regular save: You save your document once
  • Git save: You save your document + remember exactly what changed + can go back to any previous version + share with others + merge changes from different people

üé® Git Workflow (The Simple Version!)

  1. Make changes to your files
  2. Check status: git status
  3. Stage changes: git add .
  4. Commit: git commit -m "Describe what you did"
  5. Push: git push origin main

That's it! You just used Git! üéâ

üåø Branches (Don't Panic - It's Easier Than It Sounds!)

Branches let you work on different things without messing up your main code.

Basic Branch Commands:

git checkout -b my-new-feature
Create and switch to new branch
git checkout main
Switch back to main branch
git branch
List all branches

Think of branches like parallel universes for your code!

üî• Common Git Problems (And How to Fix Them!)

üò± "I messed up my files!"

üò∞ "I committed something by mistake!"

ü§î "I forgot to add something to my commit!"

git commit --amend -m "Updated commit message"

üí° Pro Tips

  • Commit often: Small commits are better than big ones
  • Write good messages: "Fixed typo" is better than "Update"
  • Use branches: Don't be afraid to experiment
  • Ask for help: Git is confusing at first - that's normal!
